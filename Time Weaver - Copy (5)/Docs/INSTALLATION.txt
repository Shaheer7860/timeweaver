===============================================================================
                    TIME WEAVER - INSTALLATION GUIDE
                    Complete Setup Instructions
===============================================================================

SYSTEM REQUIREMENTS:
--------------------
- Operating System: Windows 10/11, Linux, or macOS
- Compiler: MinGW-w64 (Windows) or GCC 5+ (Linux/Mac)
- C++ Standard: C++11 or higher
- SQLite3: Version 3.x
- RAM: Minimum 512MB
- Disk Space: 50MB
- Browser: Chrome, Firefox, Edge, or Safari (latest version)

===============================================================================
                    WINDOWS INSTALLATION
===============================================================================

STEP 1: INSTALL MINGW-W64
--------------------------
1. Download MinGW-w64 from: https://www.mingw-w64.org/
2. Run the installer
3. Select:
   - Architecture: x86_64
   - Threads: posix
   - Exception: seh
4. Add to PATH:
   - Open System Properties â†’ Environment Variables
   - Edit PATH variable
   - Add: C:\mingw64\bin
5. Verify installation:
   Open Command Prompt and run:
   ```
   g++ --version
   ```
   Should show GCC version 5.0 or higher

STEP 2: VERIFY SQLITE3
-----------------------
SQLite3 files are included in the project:
- sqlite3.c
- sqlite3.h

No additional installation needed!

STEP 3: EXTRACT PROJECT
------------------------
1. Extract "Time Weaver - Copy (5)" to a folder
2. Recommended path: C:\Projects\TimeWeaver
3. Avoid paths with spaces or special characters

STEP 4: COMPILE PROJECT
------------------------
1. Open Command Prompt
2. Navigate to project folder:
   ```
   cd "C:\Users\HP\Desktop\Time Weaver - Copy (5)"
   ```
3. Run build script:
   ```
   build.bat
   ```
4. Wait for compilation (30-60 seconds)
5. Look for "Build successful!" message

STEP 5: RUN SERVER
------------------
1. In the same Command Prompt:
   ```
   TimeWeaver.exe
   ```
2. You should see:
   ```
   Server listening on port 8080...
   ```
3. Keep this window open

STEP 6: ACCESS APPLICATION
---------------------------
1. Open your web browser
2. Navigate to: http://localhost:8080
3. You should see the login page
4. Default admin credentials:
   - Username: admin
   - Password: admin123

===============================================================================
                    TROUBLESHOOTING
===============================================================================

PROBLEM: "g++ is not recognized"
SOLUTION:
- Windows: Ensure MinGW bin folder is in PATH
- Linux/Mac: Install GCC using package manager

PROBLEM: "Cannot open database"
SOLUTION:
- Ensure you have write permissions in project folder
- Delete calendar.db and restart server
- Check disk space

PROBLEM: "Port 8080 already in use"
SOLUTION:
- Close other applications using port 8080
- Or modify server.cpp line with PORT definition
- Recompile after change

PROBLEM: "sqlite3.h not found"
SOLUTION:
- Ensure sqlite3.h is in project folder
- Check include path in compilation command

PROBLEM: "Compilation errors"
SOLUTION:
- Ensure C++11 or higher
- Check all source files are present
- Verify no syntax errors

PROBLEM: "Browser shows 'Cannot connect'"
SOLUTION:
- Ensure server is running
- Check firewall settings
- Try http://127.0.0.1:8080 instead

PROBLEM: "Login doesn't work"
SOLUTION:
- Check console for errors
- Verify database was created (calendar.db)
- Try admin/admin123 credentials

PROBLEM: "Events don't save"
SOLUTION:
- Check browser console for errors
- Verify server is running
- Check database file permissions

===============================================================================
                    DATABASE SETUP
===============================================================================

AUTOMATIC SETUP:
----------------
On first run, the server automatically:
1. Creates calendar.db file
2. Creates all tables
3. Populates universities (18 entries)
4. Populates departments (42 entries)
5. Builds search index

MANUAL RESET:
-------------
To reset database:
1. Stop server (Ctrl+C)
2. Delete calendar.db file
3. Restart server
4. Database will be recreated


===============================================================================
                    CONFIGURATION
===============================================================================

PORT CONFIGURATION:
-------------------
To change port (default: 8080):
1. Open server.cpp
2. Find line: int PORT = 8080;
3. Change to desired port
4. Recompile: build.bat
5. Access: http://localhost:NEW_PORT

DATABASE LOCATION:
------------------
Default: ./calendar.db (same folder as executable)
To change:
1. Open database.cpp
2. Find: openDatabase()
3. Modify path
4. Recompile

CACHE SIZE:
-----------
Default LRU cache: 100 entries
To change:
1. Open database.cpp
2. Find: new LRUCache(100)
3. Change capacity
4. Recompile


===============================================================================
                    PERFORMANCE TUNING
===============================================================================

FOR BETTER PERFORMANCE:
-----------------------
1. Use -O3 optimization flag:
   g++ -std=c++11 -O3 ...

2. Enable database indexing (already done)

3. Increase LRU cache size for more users

4. Use SSD for database file

5. Close unnecessary applications


===============================================================================
                    BACKUP & RESTORE
===============================================================================

BACKUP:
-------
1. Stop server
2. Copy calendar.db to backup location
3. Restart server

RESTORE:
--------
1. Stop server
2. Replace calendar.db with backup
3. Restart server

EXPORT DATA:
------------
Use SQLite tools:
```bash
sqlite3 calendar.db .dump > backup.sql
```

IMPORT DATA:
------------
```bash
sqlite3 calendar.db < backup.sql
```


===============================================================================
                    UNINSTALLATION
===============================================================================

WINDOWS:
--------
1. Stop server (Ctrl+C)
2. Delete project folder
3. Delete calendar.db (if outside project folder)
4. (Optional) Remove MinGW if not needed

LINUX/MAC:
----------
1. Stop server (Ctrl+C)
2. rm -rf "Time Weaver - Copy (5)"
3. rm calendar.db (if outside project folder)
