===============================================================================
                        DSA IMPLEMENTATIONS (10 Total)
===============================================================================

1. TRIE (Prefix Tree)
   - File: dsa_structures.h (lines 26-77)
   - Purpose: Fast event search with prefix matching
   - Complexity: O(m + k) search, O(n * m) space
   - Use: Event search, autocomplete suggestions

2. PRIORITY QUEUE (Min Heap)
   - File: dsa_structures.h (lines 79-139)
   - Purpose: Event scheduling and sorting
   - Complexity: O(log n) insert, O(1) peek
   - Use: Next event retrieval, priority management

3. LRU CACHE (Doubly Linked List + Hash Map)
   - File: dsa_structures.h (lines 141-208)
   - Purpose: Holiday data caching
   - Complexity: O(1) get/put
   - Use: Reduce database queries, improve performance

4. RECURRENCE ENGINE (Date Algorithms)
   - File: dsa_structures.h (lines 210-332)
   - Purpose: Calculate recurring event dates
   - Complexity: O(1) per check
   - Use: Daily/weekly/monthly events, birthdays

5. COURSE GRAPH (Directed Acyclic Graph)
   - File: dsa_structures.h (lines 334-419)
   - Purpose: Course prerequisite tracking
   - Complexity: O(V + E) topological sort
   - Use: Dependency resolution, cycle detection
   - Algorithms: Kahn's Algorithm, BFS

6. BINARY SEARCH TREE
   - File: dsa_structures.h (lines 421-492)
   - Purpose: Time-based event queries
   - Complexity: O(log n) insert, O(log n + k) range query
   - Use: Find events in time range, conflict detection

7. HASH MAPS (Unordered Map)
   - Used in: Multiple classes
   - Purpose: Fast lookups
   - Complexity: O(1) average
   - Use: User management, department lookup, Trie nodes

8. SORTING ALGORITHMS
   - File: database.cpp
   - Purpose: Event ordering by time
   - Complexity: O(n log n)
   - Algorithm: std::sort (Introsort)

9. REGEX PATTERN MATCHING
   - File: database.cpp
   - Purpose: Input validation
   - Complexity: O(m)
   - Use: Roll number validation, department extraction

10. QUEUE (FIFO)
    - Used in: CourseGraph
    - Purpose: BFS traversal
    - Complexity: O(1) enqueue/dequeue
    - Use: Topological sort, level-order processing
